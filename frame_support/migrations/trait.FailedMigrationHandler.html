Handles a failed runtime migration.

Required Methods§

Source

fn failed(migration: Option<u32>) -> FailedMigrationHandling

Infallibly handle a failed runtime migration.

Gets passed in the optional index of the migration in the batch that caused the failure. Returning None means that no automatic handling should take place and the callee decides in the implementation what to do.

Dyn Compatibility§

This trait is not dyn compatible.

In older versions of Rust, dyn compatibility was called "object safety", so this trait is not object safe.
